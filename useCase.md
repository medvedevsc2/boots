# 2 Варианты использования

<a name="register_login"/>

## 2.1 Зарегистрироваться / Войти

**Описание.** Этот вариант использования позволяет клиенту создать новую учетную запись или войти в существующую.  
**Предусловия.** Клиент находится на главной странице магазина.  
**Основной поток.**
1. Система отображает опции "Зарегистрироваться" и "Войти".
2. Клиент выбирает одну из опций:
    * **Для регистрации:**
        1. Клиент вводит необходимые данные (имя, электронную почту, пароль).
        2. Система проверяет данные и создает новую учетную запись.
        3. Система автоматически входит в учетную запись клиента.
    * **Для входа:**
        1. Клиент вводит свои учетные данные (логин и пароль).
        2. Система проверяет учетные данные.
        3. Система предоставляет доступ к учетной записи клиента.
3. Вариант использования завершается.

**Альтернативный поток А1 (Некорректные данные).**
1. Система отображает сообщение о некорректных данных (неверный пароль, уже существующая электронная почта).
2. Система предлагает клиенту повторить ввод данных или восстановить пароль.

**Постусловия.** Клиент авторизован в системе магазина кроссовок.

<a name="view_available_sneakers"/>

## 2.2 Просмотреть доступные кроссовки

**Описание.** Клиент может просматривать список всех кроссовок, доступных для покупки.  
**Предусловия.** Клиент находится в системе магазина.  
**Основной поток.**
1. Клиент выбирает опцию "Просмотреть доступные кроссовки".
2. Система отображает список всех доступных кроссовок с базовой информацией (бренд, модель, цена, размеры).
3. Вариант использования завершается.

**Постусловия.** Клиент видит актуальный список товаров.

<a name="search_sneakers_by_parameters"/>

## 2.3 Искать кроссовки по параметрам

**Описание.** Клиент может осуществлять поиск кроссовок по различным параметрам.  
**Предусловия.** Клиент просматривает список доступных кроссовок.  
**Основной поток.**
1. Клиент выбирает опцию "Искать кроссовки по параметрам".
2. Система отображает поля для ввода параметров поиска (бренд, размер, ценовой диапазон, цвет).
3. Клиент вводит желаемые параметры поиска.
4. Система фильтрует и отображает кроссовки, соответствующие заданным параметрам.
5. Вариант использования завершается.

**Альтернативный поток А2 (Нет результатов).**
1. Система отображает сообщение об отсутствии результатов, соответствующих заданным параметрам.
2. Система предлагает клиенту изменить параметры поиска.

**Постусловия.** Клиент видит отфильтрованный список товаров.

<a name="add_sneakers_to_cart"/>

## 2.4 Добавить кроссовки в корзину

**Описание.** Клиент может добавлять выбранные кроссовки в свою виртуальную корзину.  
**Предусловия.** Клиент просматривает информацию о конкретных кроссовках.  
**Основной поток.**
1. Клиент выбирает желаемые кроссовки (модель, размер).
2. Клиент нажимает кнопку "Добавить в корзину".
3. Система добавляет кроссовки в корзину клиента.
4. Система уведомляет клиента об успешном добавлении товара в корзину.
5. Вариант использования завершается.

**Альтернативный поток А3 (Нет в наличии).**
1. Система отображает сообщение о том, что выбранных кроссовок нет в наличии.
2. Система предлагает клиенту просмотреть похожие товары.

**Постусловия.** Выбранные кроссовки добавлены в корзину клиента.

<a name="place_order"/>

## 2.5 Оформить заказ

**Описание.** Клиент может оформить заказ на товары, находящиеся в его корзине.  
**Предусловия.** В корзине клиента есть хотя бы один товар.  
**Основной поток.**
1. Клиент переходит в свою корзину.
2. Клиент проверяет список товаров и их количество.
3. Клиент нажимает кнопку "Оформить заказ".
4. Система запрашивает информацию о доставке и оплате.
5. Клиент вводит необходимую информацию.
6. Система подтверждает создание заказа и предоставляет номер заказа.
7. Вариант использования завершается.

**Альтернативный поток А4 (Ошибка оплаты/доставки).**
1. Система отображает сообщение об ошибке при обработке оплаты или невозможности доставки по указанному адресу.
2. Система предлагает клиенту повторить ввод данных или изменить способ оплаты/доставки.

**Постусловия.** Заказ клиента создан и ожидает обработки.

<a name="view_order_history"/>

## 2.6 Просмотреть историю заказов

**Описание.** Клиент может просматривать список всех своих ранее оформленных заказов.  
**Предусловия.** Клиент авторизован в системе и имеет хотя бы один оформленный заказ.  
**Основной поток.**
1. Клиент выбирает опцию "Просмотреть историю заказов".
2. Система отображает список всех заказов клиента с основной информацией (номер заказа, дата, статус).
3. Клиент может выбрать конкретный заказ для просмотра детальной информации.
4. Вариант использования завершается.

**Постусловия.** Клиент видит историю своих покупок.

3. Менеджер заполняет форму и нажимает кнопку "Сохранить".
4. Система добавляет новые кроссовки в каталог.

**Альтернативный поток А5 (Некорректные данные).**

2. Система предлагает менеджеру повторить ввод.

**Постусловия.** Новые кроссовки добавлены в ассортимент магазина.

## 2.8 Изменить информацию о кроссовках

**Описание.** Менеджер может изменять существующую информацию о кроссовках в каталоге.

**Основной поток.**
1. Менеджер выбирает опцию "Изменить информацию о кроссовках".
2. Менеджер выбирает кроссовки, информацию о которых нужно изменить.
3. Система отображает текущую информацию о кроссовках в редактируемой форме.
4. Менеджер вносит необходимые изменения и нажимает кнопку "Сохранить".
5. Система обновляет информацию о кроссовках в каталоге.
6. Вариант использования завершается.

**Альтернативный поток А6 (Некорректные данные).**
1. Система отображает сообщение о некорректно введенных данных.
2. Система предлагает менеджеру повторить ввод.

**Постусловия.** Информация о кроссовках обновлена в каталоге.

<a name="delete_sneakers"/>

## 2.9 Удалить кроссовки

**Описание.** Менеджер может удалять кроссовки из каталога магазина.  
**Предусловия.** Менеджер авторизован в системе.  
**Основной поток.**
1. Менеджер выбирает опцию "Удалить кроссовки".
2. Менеджер выбирает кроссовки, которые нужно удалить.
3. Система запрашивает подтверждение удаления.
4. Менеджер подтверждает удаление.
5. Система удаляет кроссовки из каталога.
6. Вариант использования завершается.

**Альтернативный поток А7 (Кроссовки в активном заказе).**
1. Система отображает сообщение о том, что кроссовки не могут быть удалены, так как они присутствуют в активных заказах.
2. Система предлагает менеджеру сначала обработать соответствующие заказы.

**Постусловия.** Кроссовки удалены из ассортимента магазина.

**Предусловия.** Менеджер авторизован в системе.  
**Основной поток.**
1. Менеджер выбирает опцию "Просмотреть все заказы".
2. Система отображает список всех заказов с основной информацией (номер заказа, клиент, дата, статус, состав заказа).

## 2.11 Изменить статус заказа

**Описание.** Менеджер может изменять статус обработки заказа клиента.  
**Предусловия.** Менеджер авторизован в системе и просматривает список заказов.  
**Основной поток.**
1. Менеджер выбирает заказ, статус которого нужно изменить.
2. Система отображает текущий статус заказа и доступные опции для изменения статуса ("В обработке", "Собран", "Отправлен", "Доставлен", "Отменен").
3. Менеджер выбирает новый статус.
4. Система обновляет статус заказа.
5. Вариант использования завершается.

**Постусловия.** Статус заказа клиента обновлен.
